|Backend: SQLite

The package actually ‘wraps’ a SQLite data base. Databases are very useful for managing large amounts of data, especially when the data consist of several ‘tables’ that are related to one another in various ways.

The dbplyr package provides a very convenient way to work with databases, especially if you know a little R and ‘tidyverse’, but only some SQL.

library(dbplyr)

6.2 Using dbplyr to access the [EnsDb.Hsapiens.v75] database

R connects to SQL databases use ‘connections’. Various packages provide interfaces that connect to a number of different databases, e.g., RSQLite, RMariaDB, … The databases can be local, or hosted on a server.

We obtain a connection to the database underlying our EnsDb object using dbconn(); usually one would use the dbConnect() method defined in the corresponding

dbconn(EnsDb.Hsapiens.v75)
## <SQLiteConnection>
##   Path: /Users/ma38727/Library/R/4.0/Bioc/3.11/library/EnsDb.Hsapiens.v75/extdata/EnsDb.Hsapiens.v75.sqlite
##   Extensions: TRUE

Use src_dbi() to access the database connection from dbplyr

tbls <- src_dbi( dbconn( EnsDb.Hsapiens.v75 ) )
tbls
## src:  sqlite 3.30.1 [/Users/ma38727/Library/R/4.0/Bioc/3.11/library/EnsDb.Hsapiens.v75/extdata/EnsDb.Hsapiens.v75.sqlite]
## tbls: chromosome, entrezgene, exon, gene, metadata, protein, protein_domain,
##   tx, tx2exon, uniprot

The output shows the location of the sqlite database, as well as the tables present in the database.
